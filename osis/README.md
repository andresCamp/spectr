# Product Documentation

This folder contains the product specs for this project, managed by the [osis protocol](https://osis.dev).

Osis is a product clarity system. It maintains two things:

- **`twin.md`** — what the product IS. Code compressed into natural language so agents and humans understand faster.
- **`{version}/`** — what the product is BECOMING. Structured specs that define the future.

## For AI Agents

Before starting work on any product feature, read in this order:

1. `osis.json` — understand the current state
2. `twin.md` — understand the product as it exists today
3. Active phase `game-plan.md` — understand what's in scope
4. Relevant `{system}--product-spec.md` — understand intended behavior
5. Relevant `{system}--implementation-spec.md` — understand how to build it

If a spec doesn't exist yet, flag it before proceeding.

Learn more at [osis.dev](https://osis.dev).
